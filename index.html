HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • h1, h2, h3: headings blocks
  • img for images, use a anchor and src link to include the photo like: img src="./assets/bowtie-cat.png" alt="Profile image of cat wearing a bow tie."
  • ol, ul, li: ordered list, unordered list, and list contents

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Use special character * to apply styling to all elements
  • Can group up elements with similar styling such as h1, h2{styling code} and that styling will apply to both header 1 and header 2
  • Inline: adds CSS styling directly to HTML file like: h1 style="color:red;" Prework Study Guide/h1
  • Internal: embeds style sheet directly to HTML file like: style h1 { color: red; } /style"
  • External: a seperate CSS styling file linked in head element with:link rel="stylesheet" href="./assets/style.css"

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Always work on a feature branch never code on main branch
  • touch file.filetype creates a new file in current dir
  • git status also shows any changes that need to be pushed on current branch
  • Merge branches with main branch after reviewed
  • Always commit with messages for version control and readability with git commit -m "message"

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Link the same way we linked the styling sheet except down after the footer
